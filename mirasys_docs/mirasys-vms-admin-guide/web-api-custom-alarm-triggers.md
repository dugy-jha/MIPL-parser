Web API Custom Alarm Triggers
Alarm trigger for Mirasys Web API

When using Mirasys Alarm API to integrate into our system, System Administrators can create custom alarm triggers in System Manager > VMS Servers > Alarm Triggers.

Under this tab, you can view all custom alarm triggers created in table format displayed by Trigger name, type, Unique ID, and Camera.

When you Enable alarm triggering API in the Alarm Triggers window, you will see all alarm triggers created from Mirasys Alarm API.

Create an alarm trigger

To create an alarm, go to VMS Servers, and select recorder.

Open Alarm Triggers.

Click the green + sign at the bottom-left of the window to Add an API trigger. A new dialog to Add new alarm trigger will open.

Select trigger type Mirasys Alarm API.

Write the Trigger Name in the free text field.

Select Type, which is either Generic Alarm or Channel.

For Channel, please define the Channel range.

Click Save. The system will generate a unique ID for the trigger, and it will be displayed in the Alarm Triggers table.

Click the green checkmark to Save the Alarm Triggers settings.

Edit an alarm trigger

To modify an alarm trigger, go to VMS Servers, and select recorder. Under Alarms, open Alarm Triggers.

Select the trigger you want to modify from the list.

Click the Modify button.

You can change the trigger name and select Channel or Generic Alarm. For Channel, please define the Channel range. The system will generate a unique ID for the trigger.

Click Save Changes.

Delete an alarm trigger

To delete an alarm trigger, go to VMS Servers, and select recorder. Under Alarms, open Alarm Triggers.

Select an alarm from the trigger list.

Click the Delete button.

Confirm the deletion in the confirmation modal.
